1. Introducción y primeros pasos

ImportApp es una aplicación portátil para gestionar un negocio de importación y reventa. Funciona directamente en el navegador — no necesita internet, ni instalación, ni servidor. Abre el archivo index.html en Google Chrome o Microsoft Edge y listo.

Recomendación: Usa siempre Chrome o Edge para tener acceso a todas las funcionalidades, incluyendo guardar datos directamente en una carpeta de tu computadora.

Registra y gestiona todos tus productos con tracking individual de estado.

Carpeta de datos (guardar en disco)

Esta funcionalidad permite que la app guarde los datos directamente como archivos JSON en tu computadora, independientemente del navegador.

  1. Ve a Configuración desde el menú lateral
  2. En la sección "Carpeta de datos", haz clic en Seleccionar carpeta de datos
  3. Elige o crea una carpeta dedicada (ejemplo: C:\Datos\ImportApp)
  4. El navegador pedirá confirmar el permiso de acceso — acepta
  5. Los datos existentes se migran automáticamente a esa carpeta

A partir de ese momento, cada cambio se escribe automáticamente. Los archivos generados son:

ArchivoContenido

imp_importaciones.jsonPedidos Amazon
imp_clientes.jsonClientes
imp_ventas.jsonPedidos de clientes
imp_abonos.jsonAbonos registrados
imp_config.jsonConfiguración del negocio
Próxima vez que abras la app: el navegador mostrará un banner pidiendo confirmar el acceso a la carpeta. Haz clic en "Permitir" y la app cargará los datos desde los archivos automáticamente.

GitHub — sincronización en la nube

Conecta la app a un repositorio privado de GitHub para acceder a tus datos desde cualquier dispositivo. Los datos se sincronizan automáticamente al hacer cualquier cambio (crear, editar, borrar) y se descargan al abrir la app.

Requisitos previos

  • Cuenta en github.com (gratuita)
  • Un repositorio privado creado (ej: importapp-datos)
  • Un Personal Access Token con permiso repo
    GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)

Pasos de configuración

  1. Ve a Configuración → 🐙 GitHub
  2. Ingresa tu usuario de GitHub, el nombre del repositorio y el branch (main)
  3. Pega tu Personal Access Token en el campo correspondiente
  4. Haz clic en 🔍 Probar conexión — verás un mensaje de éxito si todo es correcto
  5. Haz clic en 💾 Guardar configuración
  6. La próxima vez que abras la app, los datos se cargarán desde GitHub automáticamente
Desde ese momento: cada vez que crees, edites o borres un registro, la app esperará 2.5 segundos y subirá automáticamente los datos a GitHub. El botón Sincronizar del menú lateral mostrará ⏳ Guardando... y luego ✅ Guardado.

Usar en otro dispositivo

  1. Abre importapp_bundle.html en el nuevo dispositivo
  2. Ve a Configuración → 🐙 GitHub e ingresa las mismas credenciales
  3. Cierra y vuelve a abrir la app — se descargará toda la data de GitHub automáticamente
Nota sobre imágenes: Las fotos de productos no se sincronizan con GitHub por su tamaño. Quedan guardadas localmente. Si cambias de dispositivo, las fotos no aparecerán hasta que las vuelvas a subir manualmente desde ese equipo.

Subir o bajar datos manualmente

Si necesitas forzar una sincronización puedes usar los botones en la sección GitHub de Configuración:

BotónAcciónCuándo usarlo
⬆️ Subir datos a GitHubSube todo lo local al repoPrimera vez, o si estuviste offline y quieres subir cambios acumulados
⬇️ Bajar datos de GitHubDescarga el repo y reemplaza lo localCuando otro dispositivo hizo cambios y quieres traerlos ahora mismo

Respaldo y restauración

TipoCuándo usarloIncluye fotos
Backup JSONRespaldo completo, antes de cambios importantes✅ Sí
Backup Excel (.xlsx)Revisión en Excel, respaldo liviano diario❌ No
Exportar CSVsAnálisis en hojas de cálculo por módulo❌ No

Para restaurar: usa el botón "⬆️ Restaurar" junto al tipo de respaldo correspondiente. Los datos actuales serán reemplazados.

3. Inventario

El inventario es el corazón de la app. Aquí registras cada producto que compras en Amazon. Cada producto tiene su propio estado de tracking, independiente de los pedidos Amazon.

Crear un producto

  1. Haz clic en + Nuevo Producto
  2. Sube una foto (opcional, se comprime automáticamente)
  3. Ingresa nombre y categoría
  4. En la sección Datos de compra Amazon: costo unitario, peso por unidad (lbs), unidades compradas y código de orden Amazon
  5. En Tracking individual: estado de tracking y estado en inventario
  6. En Precio de venta: ingresa el % de ganancia o el precio directo — se calculan mutuamente
  7. Guarda
Precio de venta: Al crear el producto solo tienes el costo Amazon. El costo courier/envío/impuestos se calculará automáticamente cuando recibas el pedido Amazon. El precio de venta se recalculará en ese momento.

4. Pedidos Amazon

Un pedido Amazon agrupa varios productos que vienen en el mismo paquete o envío. Sirve para calcular el peso total y distribuir los costos de courier, envío e impuestos proporcionalmente entre los productos.

Flujo de estados

📋 pedido
→
🚚 en transporte
→
✅ entregado

Crear un pedido Amazon

  1. Haz clic en + Nuevo Pedido
  2. Ingresa el nombre del pedido (ej: "Pedido Mayo 2025")
  3. Selecciona la fecha y el estado inicial
  4. En Productos en este pedido, haz clic en + Agregar producto y selecciona los productos que vienen en este paquete
  5. Por cada producto, ajusta las unidades y el peso por unidad si es diferente al registrado
  6. En Costos del pedido:
    • Tarifa courier: precio por libra que cobra el courier
    • Courier total: se calcula automáticamente (tarifa × peso total). Puedes editarlo manualmente
    • Costo de envío: cargo adicional de Amazon, flete u otro
    • Impuestos / Aduana: costos de aduana o tasas
  7. Guarda el pedido

Cálculo automático de costos

Al agregar productos y definir la tarifa, la app calcula automáticamente:

  • Peso total = suma de (peso por unidad × cantidad) de todos los productos
  • Courier total = tarifa × peso total
  • Total del pedido = Amazon + Courier + Costo envío + Impuestos

Marcar como entregado

Cuando recibes el paquete:

  1. Haz clic en ✅ Marcar entregado
  2. La app muestra un preview de cómo se distribuirán los costos entre los productos
  3. Revisa los valores calculados: costo adicional por unidad, inversión total y precio de venta de cada producto
  4. Si todo está correcto, confirma con ✅ Confirmar y actualizar stock
Al confirmar la entrega: Los costos (courier + envío + impuestos) se distribuyen proporcionalmente por peso entre todos los productos. Se actualizan los campos inversionTotal, precioVenta y stock de cada producto. Los productos pasan a estado disponible en inventario.

Fórmula de distribución

Para cada producto en el pedido:

VariableCálculo
Proporción de pesopesoProducto ÷ pesoTotalPedido
Costo adicional del productoproporciónDePeso × (courier + envío + impuestos)
Costo adicional por unidadcostoAdicionalProducto ÷ unidades
Inversión totalcostoAmazon + costoAdicionalPorUnidad
Precio de ventainversiónTotal × (1 + %ganancia ÷ 100)

6. Pedidos Clientes

Registra las ventas a tus clientes. El stock se descuenta automáticamente al confirmar cada pedido.

Crear un pedido cliente

  1. Haz clic en + Nuevo Pedido Cliente
  2. Selecciona el cliente del listado (se indica si tiene saldo pendiente)
  3. Selecciona un producto del desplegable y haz clic en + Agregar
  4. Ajusta las cantidades con los botones + / − en el carrito. El total se actualiza automáticamente
  5. En Pagado en este momento, ingresa el monto recibido (puede ser 0 si es completamente a crédito)
  6. Haz clic en ✅ Confirmar Pedido
Stock: Solo aparecen en el carrito los productos con stock disponible mayor a 0. Si un producto se agota tras el pedido, su estado cambia automáticamente a agotado.

Pagos parciales y abonos

Si el cliente no pagó el total:

  • El pedido queda con estado parcial o pendiente
  • En la tabla de pedidos, haz clic en 💵 Abonar para registrar un pago adicional
  • Cuando el saldo llega a 0, el estado cambia a pagado

Estados de un pedido cliente

EstadoSignificado
pagadoEl cliente pagó el total
parcialHubo un pago inicial pero queda saldo
pendienteNo se ha recibido ningún pago

Ver detalle de un pedido

Haz clic en el botón 👁️ para ver el desglose completo: productos, cantidades, precios, pagos y saldo actual.

7. Cartera

La cartera muestra todos los clientes que tienen saldos pendientes de pago, ordenados de mayor a menor deuda.

Resumen de métricas

  • Cartera Total: suma de todos los saldos pendientes
  • Cobrado: total recibido en todos los pedidos
  • Clientes en mora: cuántos tienen saldo mayor a 0
  • Clientes al día: cuántos tienen saldo 0

Registrar un abono desde cartera

  1. Haz clic en 💵 Abonar en la fila del cliente
  2. Selecciona a qué pedido aplicar el abono (si el cliente tiene varios con saldo)
  3. Ingresa el monto, fecha y forma de pago
  4. Confirma con 💾 Registrar Abono

Exportar cartera a Excel

Tres opciones en el encabezado de la pantalla:

BotónResultado
⬇️ En moraSolo clientes con saldo pendiente
⬇️ TodoTodos los clientes con su saldo
⬇️ Por fechas...Clientes con deuda de pedidos en un rango de fechas

8. Catálogo PDF

Genera un catálogo visual con los productos disponibles para compartir con clientes.

Opciones del catálogo

  • Título: nombre del catálogo
  • Mostrar precios: activa o desactiva los precios de venta
  • Filtro por categoría: genera el catálogo de una categoría específica

El catálogo incluye: logo del negocio, nombre, descripción, foto del producto, precio (opcional) y categoría. Los productos se organizan en una grilla de 3 columnas por página. Cada página lleva un pie con el número de página y el nombre del negocio.

Para mejores resultados: asegúrate de que los productos tengan foto y descripción antes de generar el catálogo.

Al restaurar desde Excel las fotos de productos no se recuperan (no se incluyen en el Excel). Para recuperar fotos, usa siempre el backup JSON.

11. Flujo completo del negocio

Este es el ciclo de vida típico de un artículo desde que lo compras hasta que cobras al cliente:

Paso 1 — Registro del producto

En Inventario: crea el producto con costo Amazon, peso, unidades y código de orden. El tracking queda en pedido.

Paso 2 — Seguimiento individual

Conforme avanza el envío, actualiza el tracking de cada producto: pedido → enviado → en aduana → en bodega.

Paso 3 — Pedido Amazon

En Pedidos Amazon: crea el pedido, agrega los productos del paquete, define tarifa courier, costo de envío e impuestos. La app calcula el total automáticamente.

Paso 4 — Recepción del paquete

Cuando llega el paquete, marca el pedido como entregado. La app distribuye los costos proporcionalmente y actualiza el stock e inversión de cada producto. Los productos pasan a disponible.

Paso 5 — Venta al cliente

En Pedidos Clientes: selecciona el cliente, arma el carrito con los productos disponibles y confirma. El stock se descuenta.

Paso 6 — Cobro

Si no pagó el total, registra abonos en Cartera hasta saldar la deuda.

12. Preguntas frecuentes

¿Puedo usar la app sin internet?

Sí. Una vez abierta en el navegador funciona completamente offline. Los CDNs (Chart.js, jsPDF, SheetJS) se cargan al abrir la app; si no hay internet, algunas funciones gráficas podrían no cargar en la primera apertura.

¿Qué pasa si cierro el navegador?

Los datos se guardan automáticamente en localStorage con cada cambio. No se pierde nada. Si tienes carpeta vinculada, también se escriben a disco.

¿Puedo usar la app en varios equipos?

Sí, de dos formas:

  • Con GitHub (recomendado): configura el mismo repositorio en cada dispositivo. Al abrir la app, descarga automáticamente los datos más recientes. Los cambios de un dispositivo se suben automáticamente y el otro los verá al abrir la app.
  • Con backup JSON: exporta el backup en un equipo, cópialo al otro y restáuralo desde Configuración.

¿Por qué no funciona la carpeta de datos en Firefox?

La funcionalidad de carpeta de datos usa el File System Access API, que solo está disponible en Chrome y Edge (versión 86 o superior). En Firefox, los datos se guardan solo en localStorage.

¿Qué hago si un producto se vendió más de lo que estaba en stock?

La app valida el stock antes de permitir agregar al carrito. No puedes vender más unidades de las disponibles.

¿Puedo editar o eliminar un pedido cliente ya confirmado?

Actualmente no es posible editar un pedido cliente una vez creado. Puedes registrar abonos o consultar el detalle. La posibilidad de editar/cancelar pedidos está planificada como mejora futura.

¿Las fotos ocupan mucho espacio?

La app comprime automáticamente las fotos al subirlas (máximo 500px, calidad JPEG 0.78). Aun así, localStorage tiene un límite de ~5 MB. En la pantalla de Configuración puedes ver cuánto espacio estás usando. Si llegas al límite, usa la carpeta de datos en disco.

¿Puedo tener el mismo producto en varios pedidos Amazon?

Sí. El mismo producto puede estar en distintos pedidos Amazon. Cada vez que un pedido se marque como entregado, el stock de ese producto aumenta y los costos se recalculan con los datos de ese pedido específico.

¿El catálogo PDF incluye todos los productos?

Solo los productos con estado disponible (en stock). Puedes filtrar por categoría para generar catálogos específicos.

Si configuro GitHub, ¿mis datos son privados?

Sí, siempre que uses un repositorio privado. Solo tú (y quienes tengan acceso al repo) pueden ver los archivos. El Personal Access Token se guarda únicamente en el localStorage de tu navegador y en la carpeta de datos local — nunca se sube a ningún servidor externo.

¿Qué pasa si hago cambios sin internet?

La app funciona normalmente offline. Los datos se guardan en localStorage y en la carpeta local (si está vinculada). Cuando recuperes la conexión, el botón Sincronizar enviará todos los cambios a GitHub. Puedes también usar el botón ⬆️ Subir datos a GitHub en Configuración.

¿Las fotos se sincronizan con GitHub?

No. Las imágenes de productos son demasiado grandes para la API de GitHub (podrían ser varios MB por foto). Solo se sincronizan los datos textuales. Las fotos quedan guardadas localmente (en localStorage o en la carpeta). Si cambias de dispositivo, deberás volver a subir las fotos desde ese nuevo equipo.
